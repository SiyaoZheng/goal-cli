# 经济学论文写作 TDD Checklist

> 来源：IZA DP No. 16276 — Plamen Nikolov, *Writing Tips for Crafting Effective Economics Research Papers* (2023–2024 Edition)，全文 59 页逐条分解。
>
> **TDD 思想**：每一条是一个针对稿件的「单元测试」——二元断言，只有 PASS / FAIL 两种状态。
> 工作流仿照 red → green → refactor：
> 1. **RED**：对当前稿件跑一遍相关 suite，标出所有 FAIL；
> 2. **GREEN**：只为让测试通过而修改稿件；
> 3. **REFACTOR**：在保持全部 PASS 的前提下精炼文字。
>
> **验证方式标签**：
> - `(M)` = 机器可检（搜索/正则/字数统计即可判定）
> - `(H)` = 人工判断（作者自查即可判定）
> - `(R)` = 需要读者测试（找一个真实读者验证）
>
> 共 **17 个 test suite，302 项**。各 suite 对应指南章节已标注，可回溯原文。
>
> 版本记录：v1.1 经 3 个核查 agent 对照原文逐页复核——修正 5 处实质偏差、删除 1 条无源条目、补入 8 条遗漏。v1.2 删除 20 条过程性/自报条目（无法从稿件本身判定 pass/fail 的，如"摘要在结果之后写""已同行审读"），只保留可对稿件执行的测试；条目编号保持稳定，缺号即已删。

## Suite 1 — RQ：研究问题（指南 §II）

- [ ] **RQ-001** 论文的研究问题能用一句疑问句完整陈述。(H)
- [ ] **RQ-002** 研究问题植根于经济理论或当下现实议题。(H)
- [ ] **RQ-003** 能用一句话回答「so what」——这个问题为什么重要。(H)
- [ ] **RQ-004** 答案不是显而易见的：结论有「turn heads」的潜质（反直觉、出人意料或推翻既有认知）。(H)
- [ ] **RQ-005** 问题具有新颖性，不是对已有研究的简单复制。(H)
- [ ] **RQ-006** 若问题已被研究过：能明确指出现有答案为何不能令人满意（gap ≠ 单纯没人做过）。(H)
- [ ] **RQ-007** 回答该问题所需的数据实际可获得。(M)
- [ ] **RQ-008** 问题范围适中：不宽到一篇论文答不完，也不窄到无人关心。(H)
- [ ] **RQ-009** 问题与作者的兴趣与专长对齐，能支撑长期投入。(H)
- [ ] **RQ-010** 问题至少与指南 18 种选题策略之一挂钩（现实事务/政策评估/当代谜题/争议议题/旧问题新方法/跨学科重构/人口与社会文化变迁/历史事件/社交媒体/福利改进等）。(H)
- [ ] **RQ-011** 若涉争议议题：分析客观、方法严谨、结论完全基于证据。(H)
- [ ] **RQ-012** 若重访旧问题（20–30 年前）：使用了新方法（DiD/RDD/IV/ML）或更丰富的新数据。(H)
- [ ] **RQ-013** 问题蕴含至少一个可检验假设（testable hypothesis）（该表述源自 §IV 建议 1，非 §II）。(H)
- [ ] **RQ-014** 能说出研究可能影响的人群及福利改善路径。(H)
- [ ] **RQ-016** 两个基本维度自评通过：(1) 问题重要且新颖；(2) 答案稳健——识别策略经得起推敲。(H)
- [ ] **RQ-017** 交叉学科问题（社会学/心理学/公共卫生）若被借用，已用经济学框架重新表述。(H)

## Suite 2 — TITLE：标题（指南 §X）

- [ ] **TITLE-002** 标题短小、易记。(H)
- [ ] **TITLE-003** 标题浓缩论文精髓：发现的快照 + 独特贡献的暗示。(H)
- [ ] **TITLE-004** 标题是分析性的，以缩略形式传达论文焦点（对读者的「承诺」）。(H)
- [ ] **TITLE-005** 标题聚焦单一最抓人的方面：topic / lesson / puzzle / conceptual takeaway 四选一作中心。(H)
- [ ] **TITLE-006** 标题与引言前几段互相呼应：引言自然展开标题的内容。(H)
- [ ] **TITLE-007** 引言兑现了标题的承诺——读者感到标题在引言中被充分展开。(R)
- [ ] **TITLE-008** 未使用 "Evidence from..." 等公式化陈词（Hamermesh 批评的模板句式）。(M)
- [ ] **TITLE-009** 修辞技巧（押韵/幽默/典故）若有使用，克制而不 gimmicky。(H)
- [ ] **TITLE-011** 标题有区分度和记忆点，能把论文从海量论文中区分出来。(R)

## Suite 3 — ABS：摘要（指南 §III–IV）

- [ ] **ABS-002** 摘要陈述了具体研究问题。(M)
- [ ] **ABS-003** 摘要用一两句话描述了研究设计/数据。(M)
- [ ] **ABS-004** 摘要报告了关键定量结果——具体数字或幅度，而非 "found interesting results" 式含糊表述。(M)
- [ ] **ABS-005** 摘要点明贡献或意义。(H)
- [ ] **ABS-006** 摘要具体、无含糊断言（对照 Oster 2012 摘要范例）。(H)

## Suite 4 — INTRO：引言（指南 §III–IV）

### 4a. 开篇与 hook

- [ ] **INTRO-002** 开篇第一段确立研究主题的重要性及其（对经济学家的）相关性。(H)
- [ ] **INTRO-003** 强开篇段遵循结构：重要性/相关性 → 具体研究问题 → 简明研究设计 + 关键发现。(H)
- [ ] **INTRO-004** 前两段内点燃读者兴趣（强开篇段或 hook 至少居其一）。(R)
- [ ] **INTRO-005** 若使用 hook：属于四类之一——迷人谜题 / 难以解释的事实 / 有分析支撑的大胆论断 / 贴近日常生活或影响广泛的故事。(H)
- [ ] **INTRO-038** 若使用 hook：载体为引人入胜的统计数字 / 发人深省的引语 / 与主题相关的有力轶事之一。(H)
- [ ] **INTRO-006** hook 与研究方向有显式连接，不是孤立的噱头。(H)
- [ ] **INTRO-007** 若用 puzzling stylized fact 策略：事实之后紧跟本文将检验的候选解释（Karlan et al. 2019 模式）。(H)
- [ ] **INTRO-008** 无冗长背景铺垫；不存在可整段删除而不损故事线的背景段。(H)

### 4b. 问题与 punchline 的位置

- [ ] **INTRO-009** "This paper examines..."（或等价句）在第二段结束前出现。(M)
- [ ] **INTRO-010** 中心研究问题最迟在第三段被引入。(M)
- [ ] **INTRO-011** 主要发现（punchline）在前几段内亮出——论文不是悬疑小说，不设悬念。(M)
- [ ] **INTRO-012** 引言整体呈「三角/BLUF」结构：具体问题 → 最关键发现 → 背景细节。(H)
- [ ] **INTRO-013** 单一最重要的贡献在引言早期用一个简明段落陈述（做出了取舍，没有罗列一切）。(H)
- [ ] **INTRO-014** 读者读完开篇能复述：关键自变量、结果变量、以及暗含的可检验假设。(R)

### 4c. 五支柱完整性

- [ ] **INTRO-015** 支柱 1「起点」具备 wow 因素（原文：通常是多项特征的组合）：现实结果的重要性 / 问题抓人且不能被一眼看透 / 争议议题 / 影响广泛。(H)
- [ ] **INTRO-016** 支柱 2「中心问题」按时出现（见 INTRO-009/010）。(M)
- [ ] **INTRO-017** 支柱 3「既有研究」只保留对智识故事线必要的文献。(H)
- [ ] **INTRO-018** 承认前人贡献（站在巨人肩上），同时温和指出 gap——不贬低、不忽略。(H)
- [ ] **INTRO-019** 支柱 4「论文价值」有 golden ticket 段落：明确列出论文推进文献的几种方式，并说明 HOW。(H)
- [ ] **INTRO-020** 贡献陈述显式回答：提供了什么新洞见/新证据？如何推进了理解？(H)
- [ ] **INTRO-021** 支柱 5「路线图」存在：报出节数 + 各节内容一句话。(M)

### 4d. 六项建议逐条

- [ ] **INTRO-022** 具体问题被直接陈述，最好直接说出可检验假设。(H)
- [ ] **INTRO-023** 重要性被解释：未经验证的理论结果，或重大政策问题（从经济学视角）。(H)
- [ ] **INTRO-024** 呈现了该问题上的不确定性/争论，以及既有结果的范围。(H)
- [ ] **INTRO-025** 激发好奇心：挑战了直觉或基线预期（有 puzzle 或 surprise 元素）。(H)
- [ ] **INTRO-026** 具体贡献 + 新颖性被陈述：更稳健的识别（如实验）？新理论模型？此前被忽视的数据集？(H)
- [ ] **INTRO-027** 主要结果被预览，且只预览关键发现。(H)
- [ ] **INTRO-028** caveats、稳健性、扩展与次要结果不出现在引言（留给正文）。(H)
- [ ] **INTRO-029** 说明了发现与前人结果的差异及其含义。(H)
- [ ] **INTRO-030** 若结果不确定/不显著：透明陈述 + 简要解释原因。(H)
- [ ] **INTRO-031** 结果预览对外行读者也清晰可懂。(R)

### 4e. 语气与打磨

- [ ] **INTRO-032** 结构可以像新闻（不埋 lead），但语气不是新闻化/op-ed 式的。(H)
- [ ] **INTRO-033** 引言简洁；没有会劝退读者的冗长段落。(H)
- [ ] **INTRO-034** 预判并回答了读者最可能提出的问题；区分了确定/不确定/有依据的判断三类陈述。(H)
- [ ] **INTRO-035** 承认了可能的替代结论或解释。(H)
- [ ] **INTRO-036** 两个独立读者读完引言的 takeaway 一致（"cannot possibly misunderstand" 测试）。(R)

## Suite 5 — LIT：文献综述（指南 §V）

### 5a. 形式选择

- [ ] **LIT-001** 已明确决定形式：独立小节 vs 融入引言（期刊投稿强烈推荐融入）。(H)
- [ ] **LIT-002** 若独立小节：标题不是 "Literature Review"（原文转述：some view this approach as somewhat sophomoric）。(M)
- [ ] **LIT-003** 若独立小节：标题是分析性的，陈述你的综合判断（如 "Restrictive Gender Norms in India"），可选：conceptual bottom line / topical / open areas 三种取向。(H)
- [ ] **LIT-005** 若融入引言：不以他人发现的长篇讨论开场——先讲自己的工作，再关联他人。(H)

### 5b. 姿态与关系

- [ ] **LIT-006** 引用慷慨，对他人工作表述尊重。(H)
- [ ] **LIT-007** 未过度批评任何文献（潜在审稿人可能就是被批评者）。(H)
- [ ] **LIT-008** 与最相关的几篇研究做了显式区分，并对相似想法给予 credit。(H)

### 5c. 加工深度

- [ ] **LIT-009** 每篇关键文献被 gist 化：主要观点浓缩成几个词；初轮提炼时先不筛选与本研究的相关性，全部提炼完再回头组织。(H)
- [ ] **LIT-010** 文献被组织成逻辑连贯的 takeaways（主题、张力、综合），而非逐篇罗列的 annotated bibliography。(H)
- [ ] **LIT-011** 综述服务于自己的分析——每一段都在为「我的论文」铺路（readers care about YOUR research）。(H)
- [ ] **LIT-012** 综述是选择性的，不是穷尽式的。(H)
- [ ] **LIT-013** 双功能同时达成：展示对领域的熟悉度 + 传达本文贡献。(H)
- [ ] **LIT-014** 界定了议题、术语、方法与前人研究的异同。(H)

### 5d. 结构与篇幅

- [ ] **LIT-016** 篇幅约 2 个单倍行距页（20 页论文基准）。(M)
- [ ] **LIT-017** 两部分结构完整：(1) 直接相关的既有研究（可比的方法/模型/数据）；(2) 本文的独特贡献。(H)
- [ ] **LIT-018** 贡献陈述具体：新数据？新模型？创新识别？更宽或更窄的范围？如何推进？(H)
- [ ] **LIT-019** 讨论了外部效度：研究人群/制度背景差异是否能解释结果分歧。(H)
- [ ] **LIT-020** 引用以同行评审的经济学期刊为主（EconLit 收录），跨学科文献适度。(M)

## Suite 6 — DATA：数据（指南 §VI）

### 6a. 数据描述

- [ ] **DATA-001** 篇幅约 1 个单倍行距页（20 页论文基准）。(M)
- [ ] **DATA-002** 数据名称、来源、覆盖时期明确陈述。(M)
- [ ] **DATA-003** 数据结构声明：panel / cross-section / time-series。(M)
- [ ] **DATA-004** 观察单位 + 总样本量 N 明确陈述。(M)
- [ ] **DATA-005** 与研究最相关的数据特征优先描述，因变量优先。(H)
- [ ] **DATA-006** 关键变量逐一介绍。(M)

### 6b. 局限披露

- [ ] **DATA-007** 局限性披露完整：缺失变量/观测、响应率、测量问题。(H)
- [ ] **DATA-008** 优先讨论「细心读者会担心」的局限，并预告后文的 falsification/robustness 回应。(H)
- [ ] **DATA-009** 小样本或代理变量不准会威胁识别/内部效度之处被显式标记。(H)
- [ ] **DATA-010** 次要局限放脚注，不占正文。(H)
- [ ] **DATA-011** 与「检验该假设的理想数据集」做了对照，说明本数据与理想的差距。(H)

### 6c. 描述统计

- [ ] **DATA-012** 所有相关变量（结果、自变量、关键控制）报告均值 + 标准差。(M)
- [ ] **DATA-013** 有汇总统计表（summary statistics table）。(M)
- [ ] **DATA-014** 数据来源被正式引用。(M)
- [ ] **DATA-015** 设计需要时报告子组统计：treatment/control、attriters/non-attriters、pre/post。(H)
- [ ] **DATA-016** 表中变量命名清晰可读。(H)
- [ ] **DATA-017** 实证测量与理论概念的差异被注明。(H)
- [ ] **DATA-018** 描述统计中的有趣模式被点出，为后续分析提供语境和外推性判断。(H)
- [ ] **DATA-019** 必要时提供关键变量的分布图。(H)
- [ ] **DATA-020** 视觉展示用于支持稳健性检查或实证方法所依赖的假设。(H)

## Suite 7 — ID：识别策略（指南 §VII）

### 7a. 总体

- [ ] **ID-001** 篇幅约 2–3 个单倍行距页（20 页论文基准）。(M)
- [ ] **ID-002** 整节读起来是一个论证：「为什么读者应该相信这里的因果结论」。(R)
- [ ] **ID-003** 使用子标题引导读者穿过分析的不同层次（复杂分析尤其）。(M)

### 7b. 六要素之一：研究设计

- [ ] **ID-004** 研究设计明确声明：RCT / 自然实验 / RDD / IV / DiD 等。(M)
- [ ] **ID-005** 论证的是设计对本问题 + 本数据的适用性，而非解释基本方法原理（不给读者上计量课）。(H)
- [ ] **ID-006** IV 设计：说明了 fit、结果变量 Y、关键解释变量 X、工具变量分别是什么。(M)
- [ ] **ID-007** RDD 设计：说明了 cutoff 规则。(M)
- [ ] **ID-008** 实验设计：设置描述足够清晰，可支撑复现（对照 Ye et al. 2019 的处理组/会话/被试描述粒度）。(H)
- [ ] **ID-009** 随机化设计：说明随机分配如何保证关键处理变量的外生性。(H)

### 7c. 六要素之二：关键变量

- [ ] **ID-010** 因变量与核心自变量有明确定义 + 测量方式。(M)
- [ ] **ID-011** 不可观测概念（如 social capital）的代理变量被说明，且论证了其适当性。(H)

### 7d. 六要素之三：统计模型

- [ ] **ID-012** 估计方程被完整写出。(M)
- [ ] **ID-013** 关键可检验假设被声明。(M)
- [ ] **ID-014** 关键参数被指明（读者知道该盯哪个系数）。(M)
- [ ] **ID-015** 设定的合理性有论证：理论驱动或先例驱动。(H)
- [ ] **ID-016** 变量的纳入/排除理由被解释。(H)
- [ ] **ID-017** 对所依赖的假设透明（如 exclusion restrictions）。(H)
- [ ] **ID-018** 计划进行的附加检验被预告。(H)
- [ ] **ID-038** 阐述研究设计时，必要处将数据与假设连接起来（数据细节仍留在数据节）。(H)

### 7e. 六要素之四：识别假设

- [ ] **ID-019** 识别假设被显式拼出：误差项性质、外生性、IV 相关性/有效性等。(M)
- [ ] **ID-020** 为每个假设给出成立的理由或证据——结合本研究情境论证，而非教科书式罗列。(H)
- [ ] **ID-021** 承诺在结果部分进一步辩护假设（且兑现了）。(H)

### 7f. 六要素之五：威胁

- [ ] **ID-022** 四大经典威胁逐一评估：OVB、联立因果、测量误差、选择偏差。(H)
- [ ] **ID-023** 每个相关威胁有对应的反制措施：控制变量、IV、固定效应等。(H)
- [ ] **ID-024** 无法消除的短板被坦率承认，而非做无支撑的强断言。(H)

### 7g. naïve OLS 偏误自检（十项逐条）

- [ ] **ID-025** 已检查：遗漏关键回归元（影响因变量的重要变量被排除在模型外）。(H)
- [ ] **ID-026** 已检查：样本选择偏差（分析样本非随机、不能代表总体）。(H)
- [ ] **ID-027** 已检查：联立性（自变量与因变量联合决定，形成反馈回路）。(H)
- [ ] **ID-028** 已检查：识别所依赖的交互项未被遗漏。(H)
- [ ] **ID-029** 已检查：未过度依赖截面变异（应看 changes 时不用 cross-sections 推断）。(H)
- [ ] **ID-030** 已检查：比较单位正确（across units vs within units 与研究问题匹配）。(H)
- [ ] **ID-031** 已检查：结果变量测量正确（没有聚焦错误的 outcome）。(H)
- [ ] **ID-032** 已检查：应纳入的固定效应已纳入。(H)
- [ ] **ID-033** 已检查：反向因果（是 Y 在驱动 X 而非相反）。(H)
- [ ] **ID-034** 已检查：不可观测异质性与测量变量相关的问题。(H)

### 7h. 比较组与稳健性预告

- [ ] **ID-035** 关键比较组（treatment vs counterfactual）被明确讨论。(H)
- [ ] **ID-036** 比较的正当性有论证：研究设计如何捕捉/构造反事实。(H)

## 附：TDD 执行建议

| 写作阶段 | 先跑的 suite（相当于「先写测试」） |
|---|---|
| 选题 | Suite 1 (RQ) |
| 初稿前 | Suite 5 (LIT)、6 (DATA)、7 (ID) 的结构性条目 |
| 初稿后 | Suite 4 (INTRO)、8 (RES)、9–10 (TAB/FIG)、11 (CONC) |

- `(M)` 条目可脚本化：正则搜 "very|obviously|clearly|in order to|whether or not"、核对 citation↔reference 双向映射、统计段落句数等。
- `(R)` 条目是最贵的测试，留到 `(M)`、`(H)` 全绿之后再跑，一次读者测试覆盖多条。
- 指南自身的元规则：**写作是迭代的**——checklist 不是跑一遍，是每轮修订后回归测试（regression test）。
